[LlamaIndex](https://www.llamaindex.ai) connects LLMs to your data through loaders, indexes, and retrievers. Each of those typically needs a credential (LLM API key, data-source token). Authsome provides them.

## Pattern 1: pass tokens to constructors

```python
from authsome.context import AuthsomeContext
from llama_index.core import Settings
from llama_index.llms.openai import OpenAI
from llama_index.readers.github import GithubRepositoryReader, GithubClient

ctx = AuthsomeContext.create()

Settings.llm = OpenAI(api_key=ctx.auth.get_access_token("openai"))

reader = GithubRepositoryReader(
    github_client=GithubClient(ctx.auth.get_access_token("github")),
    owner="agentrhq",
    repo="authsome",
)
docs = reader.load_data(branch="main")
```

Refresh is silent. Re-call `get_access_token` if your process lives long enough to outlast a token's TTL.

## Pattern 2: run the whole script under the proxy

```bash
uvx authsome login openai
uvx authsome login github
uvx authsome run -- python my_index.py
```

LlamaIndex's HTTP clients see `OPENAI_API_KEY=authsome-proxy-managed` and similar placeholders. The proxy injects the real value for outbound requests to matched provider hosts.

## Data-loader credentials

LlamaIndex's data readers cover dozens of services (Notion, Confluence, Slack, Google Drive, etc.). For each one:

- Check `uvx authsome list` for the matching provider name.
- If bundled, log in: `uvx authsome login <provider>`.
- Pass `ctx.auth.get_access_token("<provider>")` to the reader's constructor.

For services not bundled, see [Custom providers](/guides/custom-providers).

## Multi-account

```python
docs_personal = reader_personal.load_data()
docs_work = reader_work.load_data()
```

with:

```python
ctx.auth.get_access_token("github", connection="personal")
ctx.auth.get_access_token("github", connection="work")
```

See [Multiple connections per provider](/guides/multiple-connections).
