Authsome is provider-side, not framework-side. It manages credentials for the services your agents call. Any agent that makes HTTP requests can use authsome through one of two patterns:

- **Run under the proxy.** `uvx authsome run -- <your agent>`. Headers are injected at request time. The agent's environment never contains real secrets.
- **Use the Python library.** `auth.get_access_token("provider")` to fetch a fresh token, then pass it to your client of choice.

This section has a page per agent framework with the install path, the recommended pattern, and any framework-specific notes.

## The two patterns, in one minute

### Proxy (recommended for production)

```bash
uvx authsome run -- python my_agent.py
```

The agent runs as a subprocess of authsome. A local HTTP proxy intercepts outbound requests and injects the right `Authorization` header per provider. The child process never sees the real secret because the proxy substitutes a placeholder env var like `OPENAI_API_KEY=authsome-proxy-managed`. See [Run agents with the proxy](/guides/run-agents-with-proxy).

### Library (recommended when you need control)

```python
from authsome.server.dependencies import create_auth_service

auth = create_auth_service()
token = auth.get_access_token("github", connection="default")
```

Direct call into authsome's auth layer. Returns a usable token. Refresh is silent. See [Python library](/reference/python-library).
