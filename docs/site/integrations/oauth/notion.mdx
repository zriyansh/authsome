Notion is a bundled OAuth2 provider in authsome. Notion pages, databases, and blocks within workspaces the user grants.

## At a glance

| | |
|---|---|
| Provider name | `notion` |
| Display name | Notion |
| Auth type | OAuth2 |
| Default flow | `pkce`, PKCE browser flow |
| PKCE supported | No |
| Device code supported | No |
| DCR supported | No |
| Default scopes | (none) |
| Proxy host | `api.notion.com` |
| Env var (`access_token`) | `NOTION_ACCESS_TOKEN` |
| Env var (`refresh_token`) | `NOTION_REFRESH_TOKEN` |

## Prerequisites

You need to register an OAuth app with Notion once. Create a public OAuth integration in your Notion workspace settings. Set the redirect URI to authsome's callback.

The redirect URI must be:

```text
http://127.0.0.1:7999/callback
```

This is the only callback URL authsome's PKCE flow listens on.

Dashboard: [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations).

<Note>
  Notion's standard OAuth flow does not use PKCE. For an MCP-style flow with DCR, use [notion_dcr](/integrations/oauth/notion-dcr) instead.
</Note>

## Log in

```bash
uvx authsome login notion
```

The first time, authsome opens a local form at `http://127.0.0.1:7999` to collect your `client_id` and `client_secret`. They are encrypted under your profile and reused on every subsequent login. A second browser window then opens to `https://api.notion.com/v1/oauth/authorize` for the authorization step.

Verify:

```bash
uvx authsome get notion --field status
# → connected
```

```bash
uvx authsome login notion --connection personal

```

## Use the token

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
token = ctx.auth.get_access_token("notion", connection="default")
```

<ProxyInjection envVar="NOTION_ACCESS_TOKEN" secretLabel="token" host="api.notion.com" />

### As an environment variable

```bash
eval "$(uvx authsome export notion --format env)"
```

Refresh tokens are never exported. Authsome refreshes silently.

## Override the bundled definition

```bash
uvx authsome inspect notion > ~/.authsome/providers/notion.json
# edit scopes, base_url, or anything else
uvx authsome list   # source now shows "custom" for notion
```

User-registered files always win over bundled definitions. See [Provider schema](/reference/provider-schema).
