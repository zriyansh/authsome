Postmark uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `postmark` |
| Display name | Postmark |
| Auth type | API key |
| Header | `X-Postmark-Server-Token: <key>` |
| Proxy host | `api.postmarkapp.com` |
| Env var | `POSTMARK_API_KEY` |
| Where to get a key | [https://account.postmarkapp.com/servers](https://account.postmarkapp.com/servers) |

## Get a key

Create an API key in the Postmark dashboard at [https://account.postmarkapp.com/servers](https://account.postmarkapp.com/servers).

## Log in

```bash
authsome login postmark
```

A local browser form opens at `http://127.0.0.1:7998`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
authsome get postmark --field status
# → connected
```

## Use the key

Run the agent under the proxy. The library tab is for embedding authsome inside a larger Python orchestrator.

<CodeGroup>

```bash Proxy (recommended)
authsome run -- python my_agent.py
```

```bash Environment
eval "$(authsome export postmark --format env)"
echo $POSTMARK_API_KEY
```

```python Python library
from authsome.server.dependencies import create_auth_service

auth = create_auth_service()
key = auth.get_access_token("postmark", connection="default")
```

</CodeGroup>

Under the proxy, authsome sets `POSTMARK_API_KEY=authsome-proxy-managed` in the child's environment and injects the real key into outbound requests to `api.postmarkapp.com`. The child process never sees the actual value.

## Multiple keys

<MultiConnectionsCta />

```bash
authsome login postmark --connection personal
authsome login postmark --connection team
```

## Rotate or remove the key

```bash
authsome login postmark --force          # rotate
authsome logout postmark                  # remove local credential
authsome remove postmark                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Postmark.

## Override the bundled definition

```bash
authsome inspect postmark > ~/.authsome/providers/postmark.json
# edit fields
authsome list   # source now shows "custom" for postmark
```

See [Provider schema](/reference/provider-schema) for every field.
