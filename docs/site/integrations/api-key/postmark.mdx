Postmark uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `postmark` |
| Display name | Postmark |
| Auth type | API key |
| Header | `X-Postmark-Server-Token: <key>` |
| Proxy host | `api.postmarkapp.com` |
| Env var | `POSTMARK_API_KEY` |
| Where to get a key | [https://account.postmarkapp.com/servers](https://account.postmarkapp.com/servers) |

## Get a key

Create an API key in the Postmark dashboard at [https://account.postmarkapp.com/servers](https://account.postmarkapp.com/servers).

## Log in

```bash
uvx authsome login postmark
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get postmark --field status
# → connected
```

## Use the key

<ProxyInjection envVar="POSTMARK_API_KEY" secretLabel="key" host="api.postmarkapp.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("postmark", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export postmark --format env)"
echo $POSTMARK_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login postmark --connection personal
uvx authsome login postmark --connection team
```

## Rotate or remove the key

```bash
uvx authsome login postmark --force          # rotate
uvx authsome logout postmark                  # remove local credential
uvx authsome remove postmark                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Postmark.

## Override the bundled definition

```bash
uvx authsome inspect postmark > ~/.authsome/providers/postmark.json
# edit fields
uvx authsome list   # source now shows "custom" for postmark
```

See [Provider schema](/reference/provider-schema) for every field.
