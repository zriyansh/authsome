Brevo uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `brevo` |
| Display name | Brevo |
| Auth type | API key |
| Header | `api-key: <key>` |
| Proxy host | `api.brevo.com` |
| Env var | `BREVO_API_KEY` |
| Where to get a key | [https://app.brevo.com/settings/keys/api](https://app.brevo.com/settings/keys/api) |

## Get a key

Create an API key in the Brevo dashboard at [https://app.brevo.com/settings/keys/api](https://app.brevo.com/settings/keys/api).

## Log in

```bash
uvx authsome login brevo
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get brevo --field status
# → connected
```

## Use the key

<ProxyInjection envVar="BREVO_API_KEY" secretLabel="key" host="api.brevo.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("brevo", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export brevo --format env)"
echo $BREVO_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login brevo --connection personal
uvx authsome login brevo --connection team
```

## Rotate or remove the key

```bash
uvx authsome login brevo --force          # rotate
uvx authsome logout brevo                  # remove local credential
uvx authsome remove brevo                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Brevo.

## Override the bundled definition

```bash
uvx authsome inspect brevo > ~/.authsome/providers/brevo.json
# edit fields
uvx authsome list   # source now shows "custom" for brevo
```

See [Provider schema](/reference/provider-schema) for every field.
