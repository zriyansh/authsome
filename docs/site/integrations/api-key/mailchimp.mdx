Mailchimp uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `mailchimp` |
| Display name | Mailchimp |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.mailchimp.com` |
| Env var | `MAILCHIMP_API_KEY` |
| Where to get a key | [https://us1.admin.mailchimp.com/account/api/](https://us1.admin.mailchimp.com/account/api/) |

## Get a key

Create an API key in the Mailchimp dashboard at [https://us1.admin.mailchimp.com/account/api/](https://us1.admin.mailchimp.com/account/api/).

## Log in

```bash
uvx authsome login mailchimp
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get mailchimp --field status
# → connected
```

## Use the key

<ProxyInjection envVar="MAILCHIMP_API_KEY" secretLabel="key" host="api.mailchimp.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("mailchimp", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export mailchimp --format env)"
echo $MAILCHIMP_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login mailchimp --connection personal
uvx authsome login mailchimp --connection team
```

## Rotate or remove the key

```bash
uvx authsome login mailchimp --force          # rotate
uvx authsome logout mailchimp                  # remove local credential
uvx authsome remove mailchimp                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Mailchimp.

## Override the bundled definition

```bash
uvx authsome inspect mailchimp > ~/.authsome/providers/mailchimp.json
# edit fields
uvx authsome list   # source now shows "custom" for mailchimp
```

See [Provider schema](/reference/provider-schema) for every field.
