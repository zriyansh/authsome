Zapier uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `zapier` |
| Display name | Zapier |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.zapier.com` |
| Env var | `ZAPIER_API_KEY` |
| Where to get a key | [https://zapier.com/app/developer/api-keys](https://zapier.com/app/developer/api-keys) |

## Get a key

Create an API key in the Zapier dashboard at [https://zapier.com/app/developer/api-keys](https://zapier.com/app/developer/api-keys).

## Log in

```bash
uvx authsome login zapier
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get zapier --field status
# → connected
```

## Use the key

<ProxyInjection envVar="ZAPIER_API_KEY" secretLabel="key" host="api.zapier.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("zapier", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export zapier --format env)"
echo $ZAPIER_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login zapier --connection personal
uvx authsome login zapier --connection team
```

## Rotate or remove the key

```bash
uvx authsome login zapier --force          # rotate
uvx authsome logout zapier                  # remove local credential
uvx authsome remove zapier                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Zapier.

## Override the bundled definition

```bash
uvx authsome inspect zapier > ~/.authsome/providers/zapier.json
# edit fields
uvx authsome list   # source now shows "custom" for zapier
```

See [Provider schema](/reference/provider-schema) for every field.
