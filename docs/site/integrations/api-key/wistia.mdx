Wistia uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `wistia` |
| Display name | Wistia |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.wistia.com` |
| Env var | `WISTIA_API_KEY` |
| Where to get a key | [https://account.wistia.com/account/api](https://account.wistia.com/account/api) |

## Get a key

Create an API key in the Wistia dashboard at [https://account.wistia.com/account/api](https://account.wistia.com/account/api).

## Log in

```bash
uvx authsome login wistia
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get wistia --field status
# → connected
```

## Use the key

<ProxyInjection envVar="WISTIA_API_KEY" secretLabel="key" host="api.wistia.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("wistia", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export wistia --format env)"
echo $WISTIA_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login wistia --connection personal
uvx authsome login wistia --connection team
```

## Rotate or remove the key

```bash
uvx authsome login wistia --force          # rotate
uvx authsome logout wistia                  # remove local credential
uvx authsome remove wistia                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Wistia.

## Override the bundled definition

```bash
uvx authsome inspect wistia > ~/.authsome/providers/wistia.json
# edit fields
uvx authsome list   # source now shows "custom" for wistia
```

See [Provider schema](/reference/provider-schema) for every field.
