Calendly uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `calendly` |
| Display name | Calendly |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.calendly.com` |
| Env var | `CALENDLY_API_KEY` |
| Where to get a key | [https://calendly.com/app/settings/developer](https://calendly.com/app/settings/developer) |

## Get a key

Create an API key in the Calendly dashboard at [https://calendly.com/app/settings/developer](https://calendly.com/app/settings/developer).

## Log in

```bash
uvx authsome login calendly
```

A local browser form opens at `http://127.0.0.1:7998`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get calendly --field status
# → connected
```

## Use the key

<CodeGroup>

```python Python library
from authsome.server.dependencies import create_auth_service

auth = create_auth_service()
key = auth.get_access_token("calendly", connection="default")
```

```bash Proxy
uvx authsome run -- python my_agent.py
```

```bash Environment
eval "$(uvx authsome export calendly --format env)"
echo $CALENDLY_API_KEY
```

</CodeGroup>

Under the proxy, authsome sets `CALENDLY_API_KEY=authsome-proxy-managed` in the child's environment and injects the real key into outbound requests to `api.calendly.com`. The child process never sees the actual value.

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login calendly --connection personal
uvx authsome login calendly --connection team
```

## Rotate or remove the key

```bash
uvx authsome login calendly --force          # rotate
uvx authsome logout calendly                  # remove local credential
uvx authsome remove calendly                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Calendly.

## Override the bundled definition

```bash
uvx authsome inspect calendly > ~/.authsome/providers/calendly.json
# edit fields
uvx authsome list   # source now shows "custom" for calendly
```

See [Provider schema](/reference/provider-schema) for every field.
