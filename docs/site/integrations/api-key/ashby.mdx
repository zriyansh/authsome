Ashby uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `ashby` |
| Display name | Ashby |
| Auth type | API key |
| Header | `Authorization: Basic <key>` |
| Proxy host | `api.ashbyhq.com` |
| Env var | `ASHBY_API_KEY` |
| Where to get a key | [https://app.ashbyhq.com/admin/api-keys](https://app.ashbyhq.com/admin/api-keys) |

## Get a key

Create an API key in the Ashby dashboard at [https://app.ashbyhq.com/admin/api-keys](https://app.ashbyhq.com/admin/api-keys).

## Log in

```bash
authsome login ashby
```

A local browser form opens at `http://127.0.0.1:7998`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
authsome get ashby --field status
# → connected
```

## Use the key

Run the agent under the proxy. The library tab is for embedding authsome inside a larger Python orchestrator.

<CodeGroup>

```bash Proxy (recommended)
authsome run -- python my_agent.py
```

```bash Environment
eval "$(authsome export ashby --format env)"
echo $ASHBY_API_KEY
```

```python Python library
from authsome.server.dependencies import create_auth_service

auth = create_auth_service()
key = auth.get_access_token("ashby", connection="default")
```

</CodeGroup>

Under the proxy, authsome sets `ASHBY_API_KEY=authsome-proxy-managed` in the child's environment and injects the real key into outbound requests to `api.ashbyhq.com`. The child process never sees the actual value.

## Multiple keys

<MultiConnectionsCta />

```bash
authsome login ashby --connection personal
authsome login ashby --connection team
```

## Rotate or remove the key

```bash
authsome login ashby --force          # rotate
authsome logout ashby                  # remove local credential
authsome remove ashby                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Ashby.

## Override the bundled definition

```bash
authsome inspect ashby > ~/.authsome/providers/ashby.json
# edit fields
authsome list   # source now shows "custom" for ashby
```

See [Provider schema](/reference/provider-schema) for every field.
