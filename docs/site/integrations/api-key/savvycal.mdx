SavvyCal uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `savvycal` |
| Display name | SavvyCal |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.savvycal.com` |
| Env var | `SAVVYCAL_API_KEY` |
| Where to get a key | [https://savvycal.com/account/api](https://savvycal.com/account/api) |

## Get a key

Create an API key in the SavvyCal dashboard at [https://savvycal.com/account/api](https://savvycal.com/account/api).

## Log in

```bash
uvx authsome login savvycal
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get savvycal --field status
# → connected
```

## Use the key

<ProxyInjection envVar="SAVVYCAL_API_KEY" secretLabel="key" host="api.savvycal.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("savvycal", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export savvycal --format env)"
echo $SAVVYCAL_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login savvycal --connection personal
uvx authsome login savvycal --connection team
```

## Rotate or remove the key

```bash
uvx authsome login savvycal --force          # rotate
uvx authsome logout savvycal                  # remove local credential
uvx authsome remove savvycal                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for SavvyCal.

## Override the bundled definition

```bash
uvx authsome inspect savvycal > ~/.authsome/providers/savvycal.json
# edit fields
uvx authsome list   # source now shows "custom" for savvycal
```

See [Provider schema](/reference/provider-schema) for every field.
