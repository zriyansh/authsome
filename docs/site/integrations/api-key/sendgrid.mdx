SendGrid uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `sendgrid` |
| Display name | SendGrid |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.sendgrid.com` |
| Env var | `SENDGRID_API_KEY` |
| Where to get a key | [https://app.sendgrid.com/settings/api_keys](https://app.sendgrid.com/settings/api_keys) |
| Provider docs | [Link](https://www.twilio.com/docs/sendgrid/ui/account-and-settings/api-keys) |

## Get a key

Create an API key in the SendGrid dashboard at [https://app.sendgrid.com/settings/api_keys](https://app.sendgrid.com/settings/api_keys).

## Log in

```bash
uvx authsome login sendgrid
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get sendgrid --field status
# → connected
```

## Use the key

<ProxyInjection envVar="SENDGRID_API_KEY" secretLabel="key" host="api.sendgrid.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("sendgrid", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export sendgrid --format env)"
echo $SENDGRID_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login sendgrid --connection personal
uvx authsome login sendgrid --connection team
```

## Rotate or remove the key

```bash
uvx authsome login sendgrid --force          # rotate
uvx authsome logout sendgrid                  # remove local credential
uvx authsome remove sendgrid                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for SendGrid.

## Override the bundled definition

```bash
uvx authsome inspect sendgrid > ~/.authsome/providers/sendgrid.json
# edit fields
uvx authsome list   # source now shows "custom" for sendgrid
```

See [Provider schema](/reference/provider-schema) for every field.
