Keywords Everywhere uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `keywords-everywhere` |
| Display name | Keywords Everywhere |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.keywordseverywhere.com` |
| Env var | `KEYWORDS_EVERYWHERE_API_KEY` |
| Where to get a key | [https://keywordseverywhere.com/manage-api-key.html](https://keywordseverywhere.com/manage-api-key.html) |

## Get a key

Create an API key in the Keywords Everywhere dashboard at [https://keywordseverywhere.com/manage-api-key.html](https://keywordseverywhere.com/manage-api-key.html).

## Log in

```bash
uvx authsome login keywords-everywhere
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get keywords-everywhere --field status
# → connected
```

## Use the key

<ProxyInjection envVar="KEYWORDS_EVERYWHERE_API_KEY" secretLabel="key" host="api.keywordseverywhere.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("keywords-everywhere", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export keywords-everywhere --format env)"
echo $KEYWORDS_EVERYWHERE_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login keywords-everywhere --connection personal
uvx authsome login keywords-everywhere --connection team
```

## Rotate or remove the key

```bash
uvx authsome login keywords-everywhere --force          # rotate
uvx authsome logout keywords-everywhere                  # remove local credential
uvx authsome remove keywords-everywhere                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Keywords Everywhere.

## Override the bundled definition

```bash
uvx authsome inspect keywords-everywhere > ~/.authsome/providers/keywords-everywhere.json
# edit fields
uvx authsome list   # source now shows "custom" for keywords-everywhere
```

See [Provider schema](/reference/provider-schema) for every field.
