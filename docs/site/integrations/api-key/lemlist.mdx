Lemlist uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `lemlist` |
| Display name | Lemlist |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.lemlist.com` |
| Env var | `LEMLIST_API_KEY` |
| Where to get a key | [https://app.lemlist.com/settings/integrations](https://app.lemlist.com/settings/integrations) |

## Get a key

Create an API key in the Lemlist dashboard at [https://app.lemlist.com/settings/integrations](https://app.lemlist.com/settings/integrations).

## Log in

```bash
uvx authsome login lemlist
```

A local browser form opens at `http://127.0.0.1:7999`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
uvx authsome get lemlist --field status
# → connected
```

## Use the key

<ProxyInjection envVar="LEMLIST_API_KEY" secretLabel="key" host="api.lemlist.com" />

### From a Python agent

```python
from authsome.context import AuthsomeContext

ctx = AuthsomeContext.create()
key = ctx.auth.get_access_token("lemlist", connection="default")
```

### As an environment variable

```bash
eval "$(uvx authsome export lemlist --format env)"
echo $LEMLIST_API_KEY
```

## Multiple keys

<MultiConnectionsCta />

```bash
uvx authsome login lemlist --connection personal
uvx authsome login lemlist --connection team
```

## Rotate or remove the key

```bash
uvx authsome login lemlist --force          # rotate
uvx authsome logout lemlist                  # remove local credential
uvx authsome remove lemlist                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Lemlist.

## Override the bundled definition

```bash
uvx authsome inspect lemlist > ~/.authsome/providers/lemlist.json
# edit fields
uvx authsome list   # source now shows "custom" for lemlist
```

See [Provider schema](/reference/provider-schema) for every field.
