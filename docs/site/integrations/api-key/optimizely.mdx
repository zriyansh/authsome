Optimizely uses a long-lived API key. Authsome stores the key encrypted in the local vault, injects it at request time through the proxy, and keeps it out of shell history, process listings, and environment dumps.

## At a glance

| | |
|---|---|
| Provider name | `optimizely` |
| Display name | Optimizely |
| Auth type | API key |
| Header | `Authorization: Bearer <key>` |
| Proxy host | `api.optimizely.com` |
| Env var | `OPTIMIZELY_API_KEY` |
| Where to get a key | [https://app.optimizely.com/v2/profile/api-access](https://app.optimizely.com/v2/profile/api-access) |

## Get a key

Create an API key in the Optimizely dashboard at [https://app.optimizely.com/v2/profile/api-access](https://app.optimizely.com/v2/profile/api-access).

## Log in

```bash
authsome login optimizely
```

A local browser form opens at `http://127.0.0.1:7998`. Paste the key into the masked input and submit.

<MaskedInputNote />

Verify:

```bash
authsome get optimizely --field status
# → connected
```

## Use the key

Run the agent under the proxy. The library tab is for embedding authsome inside a larger Python orchestrator.

<CodeGroup>

```bash Proxy (recommended)
authsome run -- python my_agent.py
```

```bash Environment
eval "$(authsome export optimizely --format env)"
echo $OPTIMIZELY_API_KEY
```

```python Python library
from authsome.server.dependencies import create_auth_service

auth = create_auth_service()
key = auth.get_access_token("optimizely", connection="default")
```

</CodeGroup>

Under the proxy, authsome sets `OPTIMIZELY_API_KEY=authsome-proxy-managed` in the child's environment and injects the real key into outbound requests to `api.optimizely.com`. The child process never sees the actual value.

## Multiple keys

<MultiConnectionsCta />

```bash
authsome login optimizely --connection personal
authsome login optimizely --connection team
```

## Rotate or remove the key

```bash
authsome login optimizely --force          # rotate
authsome logout optimizely                  # remove local credential
authsome remove optimizely                  # remove all local state for the provider
```

API-key providers have no revocation endpoint, so `revoke` and `remove` are equivalent for Optimizely.

## Override the bundled definition

```bash
authsome inspect optimizely > ~/.authsome/providers/optimizely.json
# edit fields
authsome list   # source now shows "custom" for optimizely
```

See [Provider schema](/reference/provider-schema) for every field.
