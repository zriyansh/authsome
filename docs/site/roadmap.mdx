## Shipped

- **CLI** — `login`, `logout`, `get`, `export`, `run`, `list`, `inspect`, `scan`, `register`, `doctor`, `whoami`. See [CLI reference](/reference/cli).
- **OAuth2 flows** — PKCE, device code, DCR + PKCE, API key with browser bridge. Automatic token refresh.
- **45 bundled providers** — 14 OAuth2 and 31 API key, plus custom provider support via `authsome register`.
- **Encrypted vault** — credentials encrypted at rest, stored locally. No cloud, no account.
- **Zero-knowledge proxy** — `authsome run` injects credentials into subprocesses without exposing them in the environment.
- **Web dashboard** — local browser UI for managing connections via `authsome ui`.
- **Audit log** — structured event log for every credential access and auth action.
- **Agent identity** — Ed25519 key pairs and Proof-of-Possession JWTs for agent-to-daemon trust.
- **Profiles** — multiple local identities with isolated credential namespaces.
- **Claude Code skill** — installable skill for Claude Code sessions.
- **Agent integrations** — Claude Code, Codex, Cursor, LangChain, LlamaIndex, OpenAI Agents SDK, Anthropic SDK. See [Integrations](/integrations/agents/index).

## Up next

- **Policy layer** — per-agent access control: which agent can use which credential on whose behalf.
- **MCP server** — native MCP tool surface so agents call authsome as a tool, not a CLI subprocess.
- **Homebrew distribution** — `brew install authsome`.
- **Multi-user support** — multiple users sharing a deployed daemon with proper isolation.

## Considering

- **Python SDK** — programmatic API for library users. [File an issue](https://github.com/agentrhq/authsome/issues) with your use case.
- **GitHub Actions integration** — first-class CI credential injection.
- **Managed SaaS** — hosted credential management for teams. No timeline yet.
- **Windows support**.

## Deliberately out of scope

- A cloud secrets manager — Authsome is local-first by design.
- Enterprise IDP replacement — federation is a downstream concern.

## Shape the roadmap

- [Open an issue](https://github.com/agentrhq/authsome/issues) to report a bug or request a feature.
- [Start a discussion](https://github.com/agentrhq/authsome/discussions) to share an idea.
- [Open a PR](https://github.com/agentrhq/authsome) — the bundled provider list is the most common contribution target.
